# Шпаргалка по Neovim

> Лидер (`<leader>`) = **Пробел** · Конфиг: `~/.config/nvim/`

---

## Основы Vim

Vim работает в **режимах**:

| Режим | Вход | Зачем |
|-------|------|-------|
| Normal | `Esc` | навигация, команды (сюда всегда возвращаешься) |
| Insert | `i` `a` `o` | печатать текст |
| Visual | `v` `V` | выделять |
| Command | `:` | команды (`:w`, `:q`) |

**Движение:** `h j k l` (←↓↑→) · `w`/`b` слово вперёд/назад · `0`/`$` начало/конец строки · `gg`/`G` начало/конец файла · `Ctrl+u`/`Ctrl+d` полэкрана

**Редактирование:** `x` символ · `dd` вырезать строку · `yy` копировать · `p` вставить · `u` отмена · `Ctrl+r` повтор · `.` повторить действие

**Поиск:** `/текст` вперёд · `n`/`N` следующее/предыдущее · `*` слово под курсором

**Файлы:** `:w` сохранить · `:q` выйти · `:wq` оба · `:q!` без сохранения

> Буфер обмена синхронён с macOS (`yy` → `Cmd+V` работает). Поиск умный без регистра.

## Твои горячие клавиши

**Поиск (Telescope)**
| `Ctrl+p` | найти файл |
| `<leader>fg` | поиск по тексту в проекте |

**Файлы / дерево**
| `Ctrl+n` | дерево файлов (Neo-tree) |
| `<leader>nf` | новый файл |
| `<leader>nd` | новая папка |

Внутри дерева: `a` создать · `d` удалить · `r` переименовать · `Enter` открыть · `q` закрыть

**Буферы (вкладки сверху, bufferline)**
| `Tab` | следующий буфер |
| `Shift+Tab` | предыдущий буфер |
| `<leader>bp` | выбрать буфер по букве |
| `<leader>bd` | закрыть буфер |

**Код / LSP**
| `K` | инфо под курсором (hover) |
| `gd` | к определению |
| `gr` | найти использования |
| `<leader>rn` | переименовать символ |
| `<leader>ca` | действия с кодом (фиксы, импорты) |

**Автодополнение (blink.cmp, в Insert)**
| `Ctrl+Space` | меню вариантов |
| `Ctrl+y` | принять |
| `Ctrl+e` | закрыть |
| `Tab`/`Shift+Tab` | по полям сниппета |

**Форматирование**
| `<leader>f` | форматнуть вручную |

> Авто-формат при `:w`: Python → ruff, Lua → stylua.

**Прочее**
| `gcc` | коммент строки · `gc` в Visual — выделенного |
| `<leader>tt` | терминал (toggleterm); `Esc` в терминале — normal |
| `<leader>m` | рендер Markdown вкл/выкл |
| `<leader>?` | эта шпаргалка (`q`/`Esc` закрыть) |

> Забыл клавишу? Жми **Пробел** и жди — which-key покажет меню.

## Команды

| `:Lazy` | плагины (`U` update, `S` sync, `q` выход) |
| `:Mason` | LSP-серверы (`i` ставить, `q` выход) |
| `:LspInfo` | активные серверы |
| `:Glow` | предпросмотр Markdown |
| `:checkhealth` | диагностика |

---

## Частые сценарии

- **Найти файл** → `Ctrl+p`, печатай имя, `Enter`
- **Найти текст в проекте** → `<leader>fg`
- **Что делает функция** → курсор на неё → `K`, `gd`
- **Python ругается / нужен импорт** → `<leader>ca`, выбери фикс
- **Отформатировать** → просто `:w` (авто) или `<leader>f`
- **Закомментить** → `gcc` (строка) или `V`+`j` затем `gc`
- **Сломался плагин** → `:Lazy sync`, потом `:checkhealth`
- **Сменить тему** → `lua/plugins/catppuccin.lua`, строка `flavour` (latte/frappe/macchiato/mocha)

## Где что лежит

```
~/.config/nvim/
├── init.lua             # точка входа, грузит lazy
├── lua/vim-options.lua  # настройки vim + общие клавиши
└── lua/plugins/         # по файлу на плагин
```

> Свой шорткат: открой файл плагина, добавь
> `vim.keymap.set('n', '<клавиша>', <действие>, {})`
