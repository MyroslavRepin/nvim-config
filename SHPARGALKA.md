## 1. Основы Vim

Vim работает в **режимах**. Это главное, что путает в начале.

| Режим | Как войти | Зачем |
|-------|-----------|-------|
| **Normal** (обычный) | `Esc` | Навигация, команды. Сюда возвращаешься всегда |
| **Insert** (ввод) | `i` `a` `o` | Печатать текст |
| **Visual** (выделение) | `v` `V` `Ctrl+v` | Выделять текст |
| **Command** (команда) | `:` | Команды типа `:w`, `:q` |

### Движение (Normal режим)
| Клавиша | Действие |
|---------|----------|
| `h` `j` `k` `l` | влево / вниз / вверх / вправо |
| `w` / `b` | следующее / предыдущее слово |
| `0` / `$` | начало / конец строки |
| `gg` / `G` | начало / конец файла |
| `{` / `}` | прыжок по абзацам |
| `Ctrl+u` / `Ctrl+d` | полэкрана вверх / вниз |
| `%` | прыжок к парной скобке |

### Вход в Insert
| Клавиша | Действие |
|---------|----------|
| `i` / `a` | вставка до / после курсора |
| `I` / `A` | вставка в начало / конец строки |
| `o` / `O` | новая строка ниже / выше |

### Редактирование
| Клавиша | Действие |
|---------|----------|
| `x` | удалить символ |
| `dd` | удалить (вырезать) строку |
| `yy` | копировать строку |
| `p` / `P` | вставить после / до |
| `cw` | заменить слово |
| `u` | отмена (undo) |
| `Ctrl+r` | повтор (redo) |
| `.` | повторить последнее действие |
| `>>` / `<<` | сдвиг строки вправо / влево |

### Поиск
| Клавиша | Действие |
|---------|----------|
| `/текст` | искать вперёд |
| `?текст` | искать назад |
| `n` / `N` | следующее / предыдущее совпадение |
| `*` | искать слово под курсором |

> Поиск без регистра, но умный (`ignorecase` + `smartcase`): `foo` найдёт `Foo`, а `Foo` — только `Foo`.

### Файлы (Command режим)
| Команда | Действие |
|---------|----------|
| `:w` | сохранить |
| `:q` | выйти |
| `:wq` / `:x` | сохранить и выйти |
| `:q!` | выйти без сохранения |

> Буфер обмена синхронён с macOS (`unnamedplus`) — `yy` копирует в системный буфер, `Cmd+V` работает везде.

## 2. Твои горячие клавиши

Это то, что **прописано лично в твоём конфиге**. Запомни в первую очередь.

### 🔍 Поиск файлов и текста (Telescope)
| Клавиша | Действие |
|---------|----------|
| `Ctrl+p` | найти файл по имени |
| `<leader>fg` (Пробел f g) | поиск по тексту во всём проекте (live grep) |

### 📁 Файловое дерево (Neo-tree)
| Клавиша | Действие |
|---------|----------|
| `Ctrl+n` | открыть дерево файлов слева |

Внутри дерева (когда фокус там):
| Клавиша | Действие |
|---------|----------|
| `Enter` | открыть файл |
| `a` | создать файл/папку |
| `d` | удалить |
| `r` | переименовать |
| `c` / `x` / `p` | копировать / вырезать / вставить |
| `q` | закрыть дерево |

### 🧠 Код / LSP
| Клавиша | Действие |
|---------|----------|
| `K` | показать инфо/доку под курсором (hover) |
| `gd` | перейти к определению (go to definition) |
| `<leader>ca` (Пробел c a) | действия с кодом (code action: фиксы, импорты) |

### ⌨️ Автодополнение (blink.cmp)
Появляется само, когда печатаешь в Insert.
| Клавиша | Действие |
|---------|----------|
| `Ctrl+Space` | открыть/закрыть меню вариантов |
| `Ctrl+y` | принять выбранный вариант |
| `Ctrl+e` | закрыть меню |
| `Ctrl+n` / `Ctrl+p` | следующий / предыдущий вариант |
| `Tab` / `Shift+Tab` | прыжок по полям сниппета |

> Источники: LSP, пути файлов, сниппеты, слова из буфера. При вызове функции показывает хинты параметров.

### 🎨 Форматирование (conform.nvim + ruff)
| Клавиша | Действие |
|---------|----------|
| `<leader>f` (Пробел f) | отформатировать файл/выделение вручную |

> **Авто-формат при сохранении** (`:w`): Python → ruff, Lua → stylua. Ruff также линтит код через LSP — ошибки видны прямо в строках.

### 🏠 Стартовый экран (snacks dashboard)
Появляется при `nvim` без файла. Жми букву:
| Клавиша | Действие |
|---------|----------|
| `n` | новый файл |
| `f` | найти файл |
| `g` | найти текст |
| `r` | недавние файлы |
| `c` | открыть конфиг |
| `p` | плагины (`:Lazy`) |
| `?` | **шпаргалка** (эта) |
| `q` | выход |

### 📖 Шпаргалка внутри nvim
| Клавиша | Действие |
|---------|----------|
| `<leader>?` (Пробел ?) | открыть эту шпаргалку в окне (`q`/`Esc` закрыть) |

### 💬 Комментарии (Comment.nvim)
| Клавиша | Действие |
|---------|----------|
| `gcc` | закомментить / раскомментить строку |
| `gc` + движение | напр. `gc3j` — коммент 3 строки вниз |
| `gc` (в Visual) | коммент выделенного |
| `gbc` | блочный комментарий |

## 3. Плагины

Что у тебя стоит (из `lazy-lock.json`), простыми словами:

| Плагин | Что делает |
|--------|------------|
| **lazy.nvim** | менеджер плагинов (грузит остальные) |
| **telescope.nvim** | поиск: файлы, текст, всё подряд (нечёткий) |
| **telescope-ui-select** | красивые выпадающие меню выбора |
| **neo-tree.nvim** | дерево файлов сбоку |
| **nvim-lspconfig** | подключает языковые серверы (автодополнение, ошибки) |
| **mason.nvim** | устанавливает LSP-серверы внутри nvim |
| **mason-lspconfig** | мост между mason и lspconfig. Авто: `lua_ls`, `pyright`, `ruff` |
| **blink.cmp** | автодополнение (popup вариантов) + сниппеты + хинты параметров |
| **friendly-snippets** | готовые сниппеты для blink |
| **conform.nvim** | форматирование кода + авто-формат при сохранении |
| **nvim-treesitter** | умная подсветка синтаксиса (lua, javascript, python) |
| **which-key.nvim** | подсказки клавиш — нажми `<leader>` и подожди, появится меню |
| **Comment.nvim** | комментирование кода (`gcc`) |
| **gitsigns.nvim** | значки git в гутере (+/~/_ слева от строк) |
| **nvim-autopairs** | авто-закрытие скобок и кавычек |
| **catppuccin** | цветовая тема (flavour: **mocha**) |
| **lualine.nvim** | строка статуса внизу (тема dracula) |
| **snacks.nvim** | стартовый дашборд (логотип, система, погода, recent, git) + утилиты |
| **indent-blankline** | вертикальные линии отступов |
| **glow.nvim** | предпросмотр Markdown в терминале (`:Glow`) |
| **nvim-web-devicons** | иконки файлов |
| **plenary / nui** | библиотеки-зависимости (сами не используются напрямую) |

### 💡 which-key — твой спасатель
Забыл клавишу? Нажми **Пробел** и жди секунду — всплывёт меню со всеми вариантами. Работает для любого префикса (`g`, `<leader>` и т.д.).

### 🌿 gitsigns — значки слева
| Знак | Значение |
|------|----------|
| `+` | добавленная строка |
| `~` | изменённая |
| `_` `‾` | удалённая |

## 4. Команды

### Плагины (lazy)
| Команда | Действие |
|---------|----------|
| `:Lazy` | открыть панель плагинов (статус, обновления) |
| `:Lazy sync` | установить/обновить/почистить |
| `:Lazy update` | обновить плагины |
| `:Lazy clean` | удалить ненужные |

Внутри `:Lazy`: `U` — update, `S` — sync, `q` — выход.

### LSP-серверы (mason)
| Команда | Действие |
|---------|----------|
| `:Mason` | менеджер LSP-серверов (UI) |
| `:LspInfo` | какие серверы активны сейчас |
| `:LspRestart` | перезапустить LSP |

Внутри `:Mason`: `i` — установить, `X` — удалить, `q` — выход.

### Treesitter
| Команда | Действие |
|---------|----------|
| `:TSUpdate` | обновить парсеры синтаксиса |

### Markdown
| Команда | Действие |
|---------|----------|
| `:Glow` | предпросмотр текущего .md файла |

### Здоровье
| Команда | Действие |
|---------|----------|
| `:checkhealth` | диагностика nvim и плагинов |

## 5. Сценарии

**Открыть проект и найти файл**
→ `Ctrl+p`, начни печатать имя, `Enter`.

**Найти где используется функция/строка**
→ `<leader>fg` (Пробел f g), введи текст.

**Походить по файлам проекта**
→ `Ctrl+n` (дерево), стрелками/`j`/`k`, `Enter` чтобы открыть.

**Понять что делает функция**
→ курсор на неё → `K` (доку), `gd` (к определению).

**Питон ругается / нужен импорт**
→ `<leader>ca` (Пробел c a) → выбери фикс из меню.

**Дополнить код / увидеть варианты**
→ просто печатай — popup сам. `Ctrl+y` принять, `Ctrl+n/p` выбрать.

**Отформатировать Python**
→ ничего не делай, сохрани `:w` — ruff форматнёт авто. Или вручную `<leader>f`.

**Закомментить кусок кода**
→ выдели `V` + `j`/`k`, потом `gc`. Или одну строку — `gcc`.

**Забыл клавишу**
→ нажми `Пробел` и жди — which-key покажет.

**Что-то сломалось с плагином**
→ `:Lazy sync`, потом `:checkhealth`.

**Сменить тему / flavour Catppuccin**
→ файл `~/.config/nvim/lua/plugins/catppuccin.lua`, строка `flavour` (latte/frappe/macchiato/mocha).

## ⚙️ Где что лежит

```
~/.config/nvim/
├── init.lua                  # точка входа, грузит lazy
├── lua/vim-options.lua       # настройки vim (leader, табы, поиск, буфер)
├── lua/plugins/              # по файлу на плагин
│   ├── telescope.lua         # Ctrl+p, <leader>fg
│   ├── neo-tree.lua          # Ctrl+n
│   ├── lsp-config.lua        # K, gd, <leader>ca + mason (lua_ls/pyright/ruff)
│   ├── blink.lua             # автодополнение (Ctrl+Space, Ctrl+y)
│   ├── conform.lua           # формат при сохранении + <leader>f
│   ├── treesitter.lua        # подсветка lua/js/python
│   ├── snacks.lua            # стартовый дашборд
│   ├── which-key.lua         # подсказки
│   └── ...                   # остальные
├── scripts/sysinfo.sh        # система для дашборда (CPU/RAM/диск)
├── SHPARGALKA.md             # эта шпаргалка (<leader>?)
└── lazy-lock.json            # зафиксированные версии плагинов
```

> Добавить свой шорткат: открой нужный файл плагина, добавь
> `vim.keymap.set('n', '<твоя клавиша>', <действие>, {})`
